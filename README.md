# ai workspace

> "정제된 맥락(Context)을 이어가야 업무 효율이 복리처럼 불어납니다."

 업무 범위가 확장됨에 따라 문서를 만드는 쪽도, 받는 쪽도 한계에 직면했습니다. 
 ai workspace는 작업의 맥락과 히스토리를 파일 기반으로 표준화하여, 누구나 AI를 통해 즉시 업무를 이어나갈 수 있는 환경을 만듭니다.
 사람이 문서로 정보를 인수인계하던 시대에서, 소스 데이터를 기반으로 AI를 통해 다시 이해하는 시대로 변하고 있습니다.

## 왜 만들었나

- **휘발되는 맥락**: AI 세션이 끝나면 맥락이 사라집니다. 같은 주제를 다룰 때마다 배경지식을 반복 주입해야 합니다.
- **증발하는 의사결정**: AI와 사람이 함께 치열하게 고민한 과정이 개인의 채팅창에만 남고 자산으로(공유가 어려움) 남지 않습니다.
- **지식 전수의 한계**: 업무량은 많아지는데 방대한 문서를 남긴들 받는 사람이 온전히 파악하기 어렵습니다.

그래서 우리는 결과물만 넘기는 게 아니라, **맥락 + 과정 + 도구**를 통째로 Git에 쌓아 AI가 읽을 수 있는 '실행 가능한 데이터'를 만듭니다.

## 시작

```bash
git clone https://kcd.codes/security/ai-workspace.git my-project
cd my-project
./start.sh
```

## 명령어

```bash
./work start "작업명"       # history/ 로그 생성 및 컨텍스트 준비
./work done  "메시지"       # 상태 갱신 + 요약 + git commit/push
./work status               # 현재 작업 상태 확인
```

## 구조

```
project/
├── AGENTS.md               # 모든 AI 도구가 작업 전 반드시 읽어야 하는 가이드
├── context/
│   ├── project_info.md     # 프로젝트 기본 정보 (불변)
│   ├── current_status.md   # 현재 진행 상황 (자동 갱신)
│   └── prompts.md          # 효과 좋았던 프롬프트/분석 방법 모음 (팀의 자산)
├── history/
│   ├── index.md            # 전체 작업 목록 (자동 갱신)
│   └── YYYY/MM/*.md        # 개별 작업 상세 기록 + 성과 포인트
├── output/                 # 외부 공유용 문서 (HTML/Confluence 호환)
└── (업무별 폴더)            # 실제 분석 소스, 정책, 설정 파일 등
```

## 예시: 침해사고 대응 (CERT)

문서를 처음부터 끝까지 읽지 않아도, AI에게 이 구조를 맡기면 바로 업무에 투입될 수 있습니다.

```
incident-response-2026/
├── context/
│   ├── project_info.md          # 대응 체계, 팀 구성, SOP
│   ├── prompts.md               # "C2 통신 패턴 추출", "페이로드 분석" 등
│   └── architecture.md          # 네트워크 구성, 자산 목록, 로그 흐름
├── history/
│   └── 2026/04/
│       ├── 2026-04-10-webshell-탐지-분석.md
│       └── 2026-04-13-c2-통신-차단.md
├── config/lambda/               # 탐지/대응 스크립트
├── evidence/                    # 로그 샘플, IOC, 분석 증거
└── output/
    └── 2026-04-13-침해사고-보고서.html
```
